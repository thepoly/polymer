# Local Development

How to get a working dev loop, plus the gotchas we keep tripping on.

## Prerequisites

- Node.js 20+
- pnpm 10
- PostgreSQL 14+ (CI uses 16; either works locally)

> Note: `mobile/` is excluded from the root `tsconfig.json` and has its own
> `package.json`. You don't need the Android SDK to work on the web app.

## First-time setup

```bash
# 1. Install root dependencies
pnpm install
# scripts/generate-env.js creates .env if missing and seeds PAYLOAD_SECRET

# 2. Edit .env to point at your local Postgres
#    DATABASE_URL=postgres://USER:PASSWORD@127.0.0.1:5432/polymer

# 3. Create the database
createdb polymer

# 4. Run migrations (CI does this via `pnpm exec payload migrate`)
pnpm exec payload migrate

# 5. Boot the dev server
pnpm dev
```

Open <http://localhost:3000> for the public site and
<http://localhost:3000/admin> for the CMS. On a fresh database, Payload
will prompt you to create the first admin user.

## Environment variables

### Required

| Var | Notes |
| --- | --- |
| `DATABASE_URL` | Primary Postgres for Payload |
| `PAYLOAD_SECRET` | Auto-generated by `scripts/generate-env.js` if `.env` is missing |

### Optional

| Var | Default | Purpose |
| --- | --- | --- |
| `LEGACY_DATABASE_URI` | unset | Read-only legacy Wagtail archive for search/archive endpoints |
| `NEXT_PUBLIC_SITE_URL` | unset | Used for absolute URLs (e.g. internal push fetch); set to `http://localhost:3000` in dev |
| `BASE_URL` | `http://127.0.0.1:3000` | Playwright base URL |
| `PLAYWRIGHT_WEB_SERVER` | `1` | Set to `0` to skip Playwright spinning up its own server |
| `PLAYWRIGHT_WEB_SERVER_COMMAND` | `pnpm dev` | What Playwright runs to bring the app up |
| `NEXT_PUBLIC_POSTHOG_KEY` / `NEXT_PUBLIC_POSTHOG_PROJECT_TOKEN` | unset | PostHog client key. Without it, analytics is fully disabled. |
| `NEXT_PUBLIC_POSTHOG_HOST` | `https://t.poly.rpi.edu` | PostHog ingest host |
| `INTERNAL_PUSH_SECRET` | unset | Shared secret between the article hook and `/api/push/send`. Without it the hook becomes a no-op. |
| `FCM_SERVICE_ACCOUNT_JSON` | unset | Raw JSON of a Firebase service account. Without it, FCM sends are no-ops. |
| `PAYLOAD_DISABLE_PUSH` | unset | Set to `1` to disable Payload's dev-mode `db.push` (run migrations manually) |

> Production sets all of these via `/var/www/polymer/shared/.env`. See
> [`deployment.md`](deployment.md).

## Common commands

| Command | What it does |
| --- | --- |
| `pnpm dev` | Next.js dev server with HMR |
| `pnpm build` | Production build (Turbopack) |
| `pnpm start` | Run the built app on `127.0.0.1:3000` |
| `pnpm lint` | ESLint over the whole repo |
| `pnpm typecheck` | `tsc --noEmit` |
| `pnpm test:smoke` | Playwright smoke suite (`tests/smoke.spec.ts`) |
| `pnpm test:search-limit` | Search query bounds test (desktop chromium) |
| `pnpm test:deploy-smoke` | Manual post-deploy smoke (`scripts/deploy-smoke.mjs`) |
| `pnpm generate:types` | Regenerate `payload-types.ts` after collection changes |
| `pnpm exec payload migrate` | Run pending migrations |
| `pnpm exec payload migrate:create <name>` | Scaffold a new migration |

## Pre-commit checklist

Run **both** before pushing:

```bash
pnpm lint
pnpm typecheck
```

CI runs these in addition to a Postgres-backed `pnpm exec payload migrate`
and `pnpm build`. Pushes that fail any step block the deploy workflow.

## Troubleshooting

### "missing secret key" on boot

Payload could not read `PAYLOAD_SECRET`. Either `.env` doesn't exist
(`scripts/generate-env.js` should have made one — check the `pnpm install`
output), or the runtime user can't read the file. In production this
usually means a wrong-user PM2 daemon; see [`deployment.md`](deployment.md).

### Payload changed my schema without a migration

In dev, Payload's Postgres adapter sets `push: true` (`payload.config.ts`),
which mutates the schema to match the config on every start. To turn this
off and force migrations only:

```bash
PAYLOAD_DISABLE_PUSH=1 pnpm dev
```

### Search returns no archive results

`LEGACY_DATABASE_URI` is unset or pointing at a database without the
expected schema. The current-content side of search will still work; the
archive side just becomes empty.

### `/api/push/send` returns 401

`INTERNAL_PUSH_SECRET` is missing or doesn't match what the article hook
is sending. Both sides read the same env var; restart the dev server after
editing `.env`.

### Android emulator can't reach the dev server

The Capacitor shell points at `poly.rpi.edu` by default. To test against a
local backend, edit `mobile/capacitor.config.ts` server.url to your
LAN-accessible host (e.g. `http://10.0.2.2:3000` for the standard Android
emulator).

### "duplicate key value" during migration

Usually the SQL migration in `scripts/run_deploy_sql_migrations.sh` ran
against a database that already received the same change via `pnpm exec
payload migrate`. The SQL script uses `IF NOT EXISTS` / `EXCEPTION WHEN
duplicate_object` guards for exactly this reason — if you see a hard
failure, double-check the new migration matches that pattern.
